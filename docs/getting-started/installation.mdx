# Installation

## 1. Download the release

Grab the latest ZIP from the [releases page](https://github.com/Kitsune-Den/KitsuneCommand/releases). The same ZIP works on both Windows and Linux.

## 2. Extract into your server's `Mods/` directory

```
7DaysToDieServer/
  Mods/
    KitsuneCommand/
      ModInfo.xml
      KitsuneCommand.dll
      KitsuneCommand.dll.config
      Config/
        Migrations/
        appsettings.json
      wwwroot/
      Plugins/
      x64/                       # Windows native libraries
        sqlite3.dll
        libSkiaSharp.dll
      linux-x64/                 # Linux native libraries
        libSkiaSharp.so
```

:::tip[Don't rename the folder]

The mod folder must be named `KitsuneCommand`. The game loads mods by folder, and other paths (database, config, plugins) are resolved relative to it.

:::

## 3. Start the server

Start your dedicated server as you normally would. On first launch, KitsuneCommand will:

- Create `<SaveGameDir>/KitsuneCommand/KitsuneCommand.db` (SQLite database)
- Run the bundled SQL migrations
- Generate admin credentials and print them to the server console

Watch the console for a line like:

```
[KitsuneCommand] Admin user created. Username: admin  Password: <random>
```

Copy those credentials somewhere safe. They're shown only once.

## 4. Open the web panel

In a browser on any machine that can reach the server, go to:

```
http://your-server-ip:8890
```

Log in with the admin credentials from step 3. You should land on the dashboard.

:::note[Port split]

The panel runs on port **8890**, not 8888. Port 8888 is the internal API, and 8890 is a lightweight frontend that handles login and proxies API calls. This split works around a compatibility issue between 7D2D's Unity/Mono runtime and OWIN's OAuth middleware.

:::
