# REST API Reference

KitsuneCommand exposes a JSON REST API on port **8888** (internal; proxied through port 8890 with the bundled frontend). Endpoints are grouped by feature area.

## Authentication

Every non-auth endpoint requires a **bearer token** obtained from [`POST /api/auth/login`](./auth.mdx). Include it as:

```
Authorization: Bearer <access_token>
```

Tokens expire after `AccessTokenExpireMinutes` (default 24h). Use `POST /api/auth/refresh` with the refresh token to rotate without re-authenticating.

## Response envelope

Every response is wrapped in an envelope:

```json
{
  "success": true,
  "data": { /* endpoint-specific payload */ },
  "message": null
}
```

On error: `success: false`, `data: null`, and `message` carries a human-readable reason. HTTP status codes follow convention (200, 400, 401, 403, 404, 500).

## Pagination

List endpoints use query parameters:

| Param | Default | Description |
|-------|---------|-------------|
| `pageIndex` | `0` | Zero-based page index |
| `pageSize` | `20` | Items per page |
| `search` | — | Optional search string |

Paginated responses wrap the result as:

```json
{
  "items": [ /* page contents */ ],
  "total": 123,
  "pageIndex": 0,
  "pageSize": 20
}
```

## Role model

Three roles, in order of permission:

- **viewer** — read-only
- **moderator** — most actions on players, tickets, economy; can't touch server admin
- **admin** — everything

Endpoint docs call out the minimum role required when it's stricter than `[Authorize]`.

## Conventions used below

- **Path column** is relative to each page's stated route prefix.
- **Request / Response** use a shorthand: `{ field, field, field }` lists top-level field names. `DtoName[]` is an array. `—` means no body.
- Admin-only endpoints are called out in the Description column.
