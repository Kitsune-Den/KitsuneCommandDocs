# Points

Per-player point balances and bulk adjustments.

## What you see

| Column | Notes |
|--------|-------|
| Player Name | |
| Points Balance | Sortable (default descending) |
| Last Sign-In | |

A search bar filters by player. Default sort is by balance descending.

## Actions

- **Adjust points.** Opens a dialog with an amount field (positive or negative) and a required **reason**. The reason is recorded for audit.

## Notes

- Points are accrued via sign-in rewards, admin adjustments, and CD key redemptions.
- Configure sign-in reward amounts in [Settings](../settings.mdx) under the Points tab.
- Negative adjustments are allowed; balances aren't clamped at zero by the adjust dialog itself.
